Mission-Critical Clarity

üåä
Dark-First, Eyes-Friendly
Deep navy backgrounds reduce eye strain during long monitoring sessions. Bright accents reserved for data that demands attention.

Do / Don't

‚úì Do

Use cyan for primary actions and interactive elements

Use monospace for ALL numeric data and coordinates

Keep status indicators visible with glow effects

Use the segmented progress bar for batch operations

Reserve red exclusively for flood alerts and destructive actions

Use subtle scanline overlays for the radar-display feel

‚úó Don't

Use white or light backgrounds ‚Äî dark-first always

Mix sans-serif fonts for data ‚Äî monospace only

Use color alone for status ‚Äî always pair with text/icon

Use rounded corners larger than 12px ‚Äî keep it technical

Add decorative gradients or shadows without purpose

Use light-mode toggles unless explicitly requested

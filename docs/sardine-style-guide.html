01

Design Principles

🛰️
Mission-Critical Clarity
Every pixel serves the operator. No decoration without purpose. Flood alerts must be unambiguous at a glance — even at 3am.
🌊
Dark-First, Eyes-Friendly
Deep navy backgrounds reduce eye strain during long monitoring sessions. Bright accents reserved for data that demands attention.
📡
Data-Dense, Not Cluttered
Operators need many data streams simultaneously. Use spatial hierarchy and color coding to layer information without overwhelming.
⚡
Monospace Precision
Coordinates, timestamps, and metrics use monospace fonts for alignment and scanability. Body text uses a clean sans-serif.

Code & Syntax

Code blocks use JetBrains Mono with semantic syntax colors mapped to the accent palette.

Python import rasterio import numpy as np def detect_flood(sar_path, threshold=-15.0): """Threshold-based flood detection from SAR backscatter.""" with rasterio.open(sar_path) as src: sigma0 = src.read(1) # σ° in dB flood_mask = sigma0 < threshold extent_km2 = np.sum(flood_mask) * src.res[0]**2 / 1e6 return flood_mask, extent_km2

12

Do / Don't

✓ Do

Use cyan for primary actions and interactive elements

Use monospace for ALL numeric data and coordinates

Use dashed borders for section dividers and nav separators

Keep corners sharp (2-4px max) for a utilitarian feel

Reserve red exclusively for flood alerts and destructive actions

Use duotone treatment on photographs and thumbnails

Use serif font for editorial/documentation prose

✗ Don't

Mix sans-serif fonts for data — monospace only

Use color alone for status — always pair with text/icon

Use rounded corners larger than 4px — keep it brutalist

Add decorative gradients or shadows without purpose

Use full-color photos — always apply duotone filter

Nest more than 2 levels of dashed + solid borders

Caution
Export at low multilook factors will produce noisier output than the on-screen display. A 3x3 spatial smooth is applied to rendered exports to compensate.
